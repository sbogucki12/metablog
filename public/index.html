This is the first in a series of articles that I will publish describing the progress as I build a new website.


Background


I had previously begun building a website that would serve as a directory to my favorite ASMR videos. That website, "Spa: ASMR", never advanced past its beginning stage and can be viewed HERE.



Plan


My plans for Spa: ASMR now exceed my original intent. I want to make it much more user-interactive. For example, I would like users to be able to log in to the site via their social media accounts, select videos and add them to personal playlists of their own favorite ASMR videos.


Whereas the original code architecture for Spa: ASMR was going to be mostly client-based, this new version is going to be built on top of its own dedicated SQL Server database, where video meta data will be stored.


Finally, the new site will have a much more modern, sleek UX compared to its current incarnation. I like sites that look like these examples:

  • Harvest

What is ASMR?


A utonomous S ensory M eridian R esponse is the "tingling" sensation that some people experience, usually in reaction to soft sounds or gentle touching.


People who experience ASMR often first notice the sensation during an exam from a doctor. Videos that try to stimulate the viewer’s ASMR have become popular on youtube. Many of these videos are in the form of role-plays; many such role-plays replicate the doctor’s exam.


Content creators on youtube who specialize in making videos to stimulate ASMR are known as "ASMR artists." Read more about ASMR on Wikipedia.


Technologies


The backend infrastructure of this new version of Spa: ASMR will be written in C#, utilizing .NET Core. The SQL Server database will be created via Entity Framework using the "code-first" methodology.


The client-side website will be written in HTML with CSS styling and JavaScript implemented via the Angular (5) framework.


When complete, the entire site will be hosted in the cloud via Microsoft Azure with a domain name purchased via GoDaddy.

Next Steps


I need to update the data model and the database via a new migration to include some constraints on the database columns. For one, some of the columns now allow for VARCHARS of maximum length, which is unnecessary. We’ll constrain these columns to a length of a more reasonable size.


Using either LINQ or SQL queries, I’m going seed the database with video data via Entity Framework (performing the Operational Relationship Mapping). During this phase, I’ll also create a class that nearly duplicates the current domain data model class. This class will serve as the public interface and interact with the CRUD controllers. The domain model class will be totally hidden from manipulation by Spa: ASMR visitors.


There is no schedule. I’m building this site among other projects. But, I hope you will join along as I continue. In the meanwhile, please follow me on social media for notifications of when I publish and check out my other sites. Thanks!
